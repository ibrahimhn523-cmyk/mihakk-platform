## 🏗️ قواعد البناء التقني

### ١ — البنية الأساسية
- **Framework:** Next.js 14+ (App Router)
- **Database:** Supabase (PostgreSQL)
- **Auth:** Supabase Auth
- **Styling:** Tailwind CSS
- **Language:** TypeScript — دائماً، لا JavaScript عادي
- **Direction:** RTL في كل مكان — dir="rtl" على الـ html

### ٢ — قاعدة tenant_id الذهبية
كل جدول في قاعدة البيانات يحمل tenant_id بلا استثناء.
كل استعلام يتضمن WHERE tenant_id = X تلقائياً.
لا تكتب استعلاماً واحداً بدون فلتر tenant_id.

### ٣ — Row Level Security (RLS)
- RLS مفعّل على كل جدول — لا استثناء
- قبل إنشاء أي جدول جديد، أنشئ سياسة RLS له فوراً
- لا تعطّل RLS لأي سبب في بيئة الإنتاج

### ٤ — الـ Subdomain
كل subdomain يُترجَم لـ tenant_id تلقائياً في middleware.ts
bare.mihakk.com  → tenant_id = 1
tamiz.mihakk.com → tenant_id = 2
admin.mihakk.com → Super Admin (بدون tenant_id)

## 🖥️ قواعد الواجهة (UI/UX)

### ١ — التجاوب مع الأجهزة (Responsive — Mobile First)
الجوال:  أقل من 640px  → تصميم مضغوط، قوائم منسدلة
التابلت: 640 إلى 1024px → تصميم وسط
الشاشة:  أكبر من 1024px → تصميم كامل مع الشريط الجانبي
اختبر كل صفحة جديدة على الأحجام الثلاثة قبل الاعتماد.

### ٢ — التخصيص لكل منشأة (tenant_branding)
كل منشأة تملك إعدادات تخصيص مستقلة تُحفظ في جدول tenant_branding:
- primary_color: اللون الرئيسي (HEX)
- secondary_color: اللون الثانوي (HEX)
- logo_url: رابط الشعار من Supabase Storage
- font_arabic: الخط العربي (Tajawal / Cairo / Noto Kufi)
- font_latin: الخط اللاتيني (Inter / Poppins / DM Sans)
- sidebar_hidden_items: القوائم المخفية (JSON)

تُطبَّق عبر CSS Variables:
--color-primary, --color-secondary, --font-arabic

### ٣ — الخطوط المدعومة
عربي: Tajawal، Cairo، Noto Kufi Arabic، IBM Plex Arabic
لاتيني: Inter، DM Sans، Poppins، Plus Jakarta Sans
كل الخطوط من Google Fonts.

### ٤ — محك نفسها قابلة للتخصيص
الألوان والشعار والخط قابلة للتغيير من admin.mihakk.com/settings

// الوصف: ماذا يفعل هذا القسم

 * وصف الدالة باختصار

- لا ملف أطول من 300 سطر
- لا تكرار — إذا كتبت نفس الكود مرتين استخرجه لدالة مشتركة

### ٤ — لا تحذف — علّق

## 🔗 طبقة التكاملات (Integration Layer)

الهيكل جاهز في src/lib/integrations/
- whatsapp: ربط فعلي مطلوب الآن (رسائل الترحيب)

- crm: هيكل جاهز — النظام يُحدَّد لاحقاً

كل تكامل يلتزم بنفس الواجهة: connect() / disconnect() / send() / verify()
